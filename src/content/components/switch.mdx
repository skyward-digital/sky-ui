## Usage

```jsx
import { Switch } from "#/ui/components/Switch"
```

```jsx
<Switch label="Switch" hideLabel />
```

## Size

The switch comes in 3 different sizes

<CodePreview client:load>
  <Switch size="sm" client:visible>
    sm
  </Switch>
  <Switch size="md" client:visible>
    md
  </Switch>
  <Switch size="lg" client:visible>
    lg
  </Switch>
  <div slot="code">
    ```jsx
    <Switch
      size="sm" // sm, md, lg
    >
      sm
    </Switch>
    ```
  </div>
</CodePreview>

## Labels

You can pass a label directly to the switch, which will automatically be associated with that switch. (be careful not to use the same label twice)

<CodePreview client:load>
  <Switch label="Switch" client:visible />
  <div slot="code">
    ```jsx
    <Switch label="Switch" />
    ```
  </div>
</CodePreview>

You can hide this label by passing the `hideLabel` prop

<CodePreview client:load>
  <Switch label="Hidden" hideLabel client:visible />
  <div slot="code">
    ```jsx
    <Switch label="Hidden" hideLabel />
    ```
  </div>
</CodePreview>

Or you can create your own labels anywhere in your app (note if `id` is used, an id will not be generated from the label)

<CodePreview client:load>
  <div className="border w-full max-w-2xl p-6 flex justify-between items-center gap-16 rounded-lg">
    <div>
      <label htmlFor="label-anywhere" className="font-semibold mb-0">
        Some form component
      </label>
      <p>Try toggling this switch</p>
    </div>
    <Switch id="label-anywhere" client:visible />
  </div>
  <div slot="code">
    ```jsx
    <div className="border w-full max-w-2xl p-6 flex justify-between items-center gap-16 rounded-lg">
      <div>
        <label htmlFor="label-anywhere" className="font-semibold mb-0">
          Some form component
        </label>
        <p>Try toggling this switch</p>
      </div>
      <Switch id="label-anywhere" />
    </div>
    ```
  </div>
</CodePreview>

When used this way, you'll need to do additional state management to make the switch work in the way you need.

### Additional detail

You can optionally pass additional detail that is displayed under the label. This does not work without a label

<CodePreview client:load>
  <Switch label="Dark Mode Example" detail="Toggle the lights" client:visible />
  <div slot="code">
    ```jsx
    <Switch label="Dark Mode Example" detail="Toggle the lights" />
    ```
  </div>
</CodePreview>

## Reverse

You can reverse the order of the label and switch by passing the `reverse` prop

<CodePreview client:load>
  <Switch label="Reversed" reverse client:visible />
  <div slot="code">
    ```jsx
    <Switch label="Reversed" reverse />
    ```
  </div>
</CodePreview>

## Customisation

### Classes

You can customize individual buttons by passing a `className` prop. If this clashes with the default styles, the `className` will take precedence.

<CodePreview client:load>
  <Switch label="Custom" className="flex flex-col" client:visible />
  <div slot="code">
    ```jsx
    <Switch label="Custom" className="flex flex-col" />
    ```
  </div>
</CodePreview>

Due to how `tailwind-merge` works, you won't need to worry about conflicting styles when using the `className` prop.
